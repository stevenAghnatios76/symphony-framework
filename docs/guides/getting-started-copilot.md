# Getting Started with Symphony + GitHub Copilot

## Prerequisites

- [GitHub Copilot](https://github.com/features/copilot) with Chat enabled
- Node.js >= 20.0.0

## Installation

1. Clone or install Symphony into your project:

```bash
npm install symphony-framework
```

2. Run the adapter translator to generate Copilot prompt files:

```bash
node node_modules/symphony-framework/adapters/copilot/translator.js
```

This creates `.github/prompts/symphony-*.prompt.md` files in your project.

## Usage

Symphony commands are available as Copilot prompt files:

```
@symphony          — Route to the right workflow (main entry point)
@symphony-brainstorm    — Multi-agent brainstorming
@symphony-create-prd    — Create product requirements
@symphony-dev-story     — Implement a user story
@symphony-quick-dev     — Fast-track simple features
```

In Copilot Chat, reference a prompt file and describe your goal. The Conductor handles routing.

## How It Works

1. You reference a `symphony-*` prompt in Copilot Chat
2. The prompt loads the Conductor (`_symphony/core/engine/conductor.xml`)
3. The Conductor detects your lifecycle phase and selects the right workflow
4. The workflow activates specialist agents with enforced disciplines
5. Agents produce artifacts following templates and checklists

## Configuration

Symphony reads from `_symphony/_config/global.yaml`. Copilot-specific settings are in `.github/copilot-instructions.md` (generated by the adapter).

## Features

- **Agent mode** — Prompt files use `mode: agent` for full agent capabilities
- **Agent memory** — Persistent sidecars track context across sessions
- **Prompt surface** — Full command surface available via prompt files
